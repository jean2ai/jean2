### Added

- **Editable file contracts** (`shared-types/file.ts`): new types for conflict-safe file editing. `EditableFileResponse` carries file content with a SHA-256 revision hash the client must echo back on save. `SaveFileRequest` (`path`, `content`, `expectedRevision`, optional `root`, `force`) and `SaveFileResponse` (`revision`, `size`, `modifiedAt`) define the save round-trip. `FileRevisionConflictDetails` describes a 409 conflict (expected vs actual revision, current content).
- **File REST client methods** (`rest/files.ts`): `readEditable(workspaceId, path, options?)` (GET `/workspaces/:id/file`) and `save(workspaceId, request, options?)` (PUT `/workspaces/:id/file`).
- **REST response types** (`types/rest-responses.ts`): `ReadEditableFileResponse` and `SaveFileResponse` aliases for the new endpoints.
- **OAuth reauthentication status**: optional `reauthRequired?: boolean` field added to `ProviderStatus`, `GmailProviderConfig`, and `ProviderStatusMessage`. The `provider.status` event in `SdkEventMap` now passes `reauthRequired` as a 5th argument, and `routeServerMessage()` emits it. This lets consumers distinguish a never-connected provider from one whose stored OAuth credentials are permanently rejected.

### Changed

- **Terminal output is buffered before listener attachment** (`TerminalConnection`): the OUTPUT opcode handler now checks `listenerCount('output')` and pushes payloads into `_preInitOutput[]` when no listeners are attached. When a consumer calls `.on('output', ...)`, `flushPreInitOutput()` replays all buffered frames. This ensures terminal output produced before subscription is not lost, enabling reliable replay for reconnecting clients. `TerminalConnection` accepts an optional `preInitOutput` constructor parameter.
