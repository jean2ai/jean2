### Added

- **`WorkspacePreconfigSettings` type** — new interface with `selectedIds: string[] | null` and `defaultId: string | null` for workspace-specific preconfig selection. Added as optional `preconfigs` field on `WorkspaceSettings`. When `selectedIds` is null or empty, all primary preconfigs are shown (backward compatible). A non-empty array filters the visible set.

### Changed

- **Agents use live preconfig** — the `Agent` type no longer carries a frozen copy of preconfig fields. Agents resolve to the live preconfig at runtime via `getPreconfig()`, eliminating the desync between preconfig edits and agent behavior.
- **Simplified `agents.promote()`** — no longer requires a `PromoteOptions` body with a tagline. Just pass the preconfig ID: `promote(id: string, options?: { signal?: AbortSignal })`.

### Removed

- **`tagline`** — removed from the `Agent` interface.
- **`agents.update()` method** — deleted from the REST namespace (it only updated tagline).
- **`PromoteOptions`, `UpdateAgentOptions`, `UpdateAgentResponse` types** — deleted from REST types.
