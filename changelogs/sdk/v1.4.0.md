### Added

- **Notification REST client**: `client.http.notifications` can read public push configuration, create or replace subscriptions, update notification preferences, and delete subscriptions.
- **Notification acknowledgement namespace**: `client.notifications.acknowledge()` sends terminal-event acknowledgements over the existing WebSocket connection.
- **Web Push contracts**: shared types now cover notification configuration, subscription input and records, preferences, event payloads, terminal statuses, and deterministic terminal event IDs.
- **Scheduled job notification opt-in**: scheduled job create, update, and response types now include `notificationsEnabled`.
- **Chat retry events**: shared protocols expose retry status, retry number, maximum retries, error type, message, delay, and retry timestamp for provider retry flows.
- **Terminal replay completion event**: terminal connections emit `replay_complete` after replayed output has been delivered, including buffering the event until a listener is attached.
- **Tool capabilities**: `ToolDefinition` can declare capabilities such as `interactive-user-input` for server-side execution filtering.
- **Cache token usage contracts**: message parts, sessions, chat usage, resumed sessions, and compaction results carry cache-read, cache-write, and non-cached token data.
- **Self-subagent policy**: preconfig contracts support `allowSelfAsSubagent`, allowing one immediate delegation to the same preconfiguration while blocking repeated ancestry use.

### Changed

- **Terminal protocol**: the `REPLAY_COMPLETE` opcode is now part of the shared terminal protocol.
- **Preconfig REST methods**: create and update requests preserve the self-delegation setting.
- **Usage compatibility**: optional cache fields on persisted message token objects remain compatible with older messages while session and event usage exposes normalized counts.
