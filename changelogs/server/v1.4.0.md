### Added

- **Web Push delivery**: the server stores browser subscriptions and sends encrypted push notifications for terminal assistant completion, failure, and permission requests using persisted VAPID credentials.
- **Notification subscription API**: authenticated REST routes expose public push configuration and support subscription creation or replacement, preference updates, and deletion without returning endpoints or encryption keys.
- **Delivery tracking and retries**: push attempts are deduplicated per event and subscription, transient network, rate-limit, and server failures are retried, and stale subscriptions are removed after permanent provider responses.
- **Cross-device suppression**: terminal pushes wait briefly for an acknowledgement from a visible, focused client displaying the matching session before dispatching, preventing redundant notifications across registered devices.
- **Scheduled job notification opt-in**: scheduled jobs persist a `notificationsEnabled` setting and include it in create, update, and scheduler responses.
- **Chat retry events and resilient retries**: retryable provider failures now use abort-aware exponential backoff with jitter, retry status events, provider circuit breaking, and side-effect protection that stops automatic replay after tool activity.
- **Tool capability filtering**: the tool catalog supports categories, capability metadata, and recommended tools. Tools requiring interactive user input are excluded from subsessions and scheduled sessions.
- **Self-subagent policy**: preconfigurations can permit one immediate self-delegation, while ancestry checks prevent repeated self-delegation and enforce allowed subagent targets and maximum depth.

### Changed

- **Permission timeout configuration**: permission requests use `JEAN2_PERMISSION_TIMEOUT_MS`, defaulting to 30 minutes with a one-minute minimum.
- **Prompt cache tracking**: provider usage now records cache-read, cache-write, and non-cached token counts on messages, sessions, chat usage events, and compaction results. OpenAI Responses requests use a session-scoped prompt cache key.
- **Terminal replay protocol**: terminal reconnects send a replay-complete frame after buffered output, allowing clients to distinguish replayed output from live output.
- **Interactive tool installation**: interactive installation hides already-installed tools, limits the visible selection list, and supports installing only catalog entries marked as recommended with `jean2 tools install --recommended`.
- **Tool repository metadata**: repository validation and resolved tool records now carry categories, capabilities, and recommended status.

### Fixed

- **Terminal replay completion**: reconnecting clients no longer need to guess when buffered terminal output has finished, avoiding duplicate or partially rendered terminal contents.
- **Subagent task validation**: resumed tasks are checked against both their parent session and preconfiguration, and invalid primary preconfigurations can no longer be used as subagents.
- **Retry failure persistence**: failed retry attempts finalize pending tool parts and persist a retry-specific assistant error without sending a duplicate terminal notification.
