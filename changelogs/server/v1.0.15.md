### Fixed

- **Windows PATH/executable stripping** — the `getToolEnv()` environment whitelist (used when the daemon spawns the server process) was missing every Windows-critical variable. `PATH` was included, but `PATHEXT` was not — so Windows could not resolve bare executable names like `git` → `git.exe`, `bun` → `bun.exe`, or `node` → `node.exe`, making them appear "not found" even though their directories were on `PATH`. Missing `SYSTEMROOT` and `COMSPEC` could also cause spawned processes to crash or refuse to start. Expanded `TOOL_SAFE_ENV_BASE` in `env.ts` to include `PATHEXT`, `SYSTEMROOT`, `WINDIR`, `COMSPEC`, `USERPROFILE`, `HOMEDRIVE`, `HOMEPATH`, `APPDATA`, `LOCALAPPDATA`, `ProgramFiles`, `ProgramFiles(x86)`, `NUMBER_OF_PROCESSORS`, `PROCESSOR_ARCHITECTURE`, and `OS`. These are system metadata (never secrets) and are no-ops on Unix where they are simply unset. The security boundary (only whitelisted keys are passed to the server) is preserved.

### Added

- **`getToolEnv()` unit tests** (`packages/server/tests/env.test.ts`) — verify Windows variables pass through, the `allowedEnv` parameter works, and that secrets (`AWS_SECRET_ACCESS_KEY`, `OPENAI_API_KEY`, etc.) are explicitly excluded from the returned environment.
