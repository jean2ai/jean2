### Fixed

- **Silent title generation failures** — `session.generate_title` now sends error responses to the client on failure instead of swallowing exceptions silently. Both the "no title generated" and exception paths notify the requesting client.
- **Unhandled throws in delete/rename** — `session.delete` and `session.rename` handlers had no try/catch. Unexpected errors (e.g., DB failures) propagated to the top-level catch and were reported to the client as `parse_error`, which was misleading. Both now have proper try/catch blocks that send operation-specific error codes (`delete_error`, `rename_error`).
- **`sessionId` on all session error responses** — fork, revert, compact, edit, delete, rename, and title generation handlers now include `sessionId` on error sends so the client can correlate errors to pending operations and clear loading states.

### Changed

- **`regenerateSessionTitle` signature** — now accepts `ws: ServerWebSocket` as a parameter so it can send error responses directly to the requesting client. The auto-title path after chat messages also passes `ws`.
