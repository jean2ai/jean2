### Added
- **HTTPS support for the built-in web client** — the bundled client (port 3774) now uses `https.createServer` when `JEAN2_TLS_ENABLED=true`, reading the same certificate and key files (`JEAN2_TLS_CERT_FILE`, `JEAN2_TLS_KEY_FILE`) as the main server. Previously the client always served plain HTTP regardless of TLS configuration, which caused insecure-context blocks on mobile browsers and broken `wss://` connections when accessing the server remotely (e.g., over Tailscale). The server now passes `JEAN2_SERVER_URL` with the correct protocol prefix (`https://` vs `http://`) so the client SDK derives the right WebSocket protocol (`wss://` vs `ws://`).
- **`jean2 open` CLI command uses HTTPS when TLS is enabled** — the open command now opens `https://localhost:3774` instead of `http://localhost:3774` when `JEAN2_TLS_ENABLED=true`, so the correct protocol is used when launching a browser to the built-in client.

### Changed
- **Client `LaunchResult.url` reflects actual protocol** — the logged URL and returned `LaunchResult` now show `https://` when TLS is enabled, matching what the client child process actually serves.
