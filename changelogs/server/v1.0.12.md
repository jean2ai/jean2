### Added

- **Generalized OAuth 2.0 + PKCE system** — new `providers/oauth-manager.ts` that handles the full OAuth lifecycle for any provider: PKCE generation, pending flow tracking, authorization code exchange, token refresh, and automatic localhost callback server. Adding a new OAuth provider (e.g., Gmail, Google Calendar) only requires `registerOAuthConfig()` + implementing `onTokensReceived()` on a `ConnectableProvider`.
- **Pluggable redirect strategies** — three strategies for handling the OAuth callback: `client_redirect` (auto-starts a localhost `Bun.serve()` on the configured port), `manual_paste` (user copies the redirect URL from the browser), and `server_callback` (for cloud-hosted servers with a public URL).
- **`POST /api/oauth/callback`** — new REST endpoint for clients to submit authorization codes. Accepts `{ flowId, code, state, redirectUri }`, validates state, exchanges the code for tokens via PKCE, and notifies the provider.
- **`GET /api/providers/:providerId/oauth/callback`** — generalized server-side OAuth callback route (for `server_callback` strategy with cloud-hosted deployments).
- **`OAuthProviderConfig` SDK type** — shared configuration interface with `clientId`, `authorizeUrl`, `tokenUrl`, `scopes`, `redirectUri`, and optional `extraAuthParams`. Lives in `@jean2/sdk` shared types so both server and client can reference it.
- **`onTokensReceived()` on `ConnectableProvider`** — providers now implement this method to save tokens after successful OAuth exchange, decoupling token storage from the flow manager.
- **Per-provider structured output modes** — new `structuredOutput.mode` model capability (`'native'` | `'prompt'`). Providers that strip the JSON schema and only support `{ type: "json_object" }` (GLM/Zhipu, MiniMax) can now be marked `prompt` mode so the schema is injected into the system prompt and the JSON is parsed from the text response instead of relying on `Output.object()`.
- **`core/structured-output.ts`** — new module with `buildSchemaPromptInstruction()` (injects the JSON Schema into the system message) and `extractJsonFromText()` (robustly extracts a JSON object from raw model output, handling markdown fences and preamble text).
- **Env presets** — new `configuration/env-presets.ts` defining well-known environment variables (`ENV_PRESETS`) with metadata: description, category, sensitivity, example, and an external documentation link. Gmail OAuth variables are seeded as the first presets.
- **`getAllJean2EnvKeys()`** — new `env.ts` export that returns all keys currently in the env overlay, used by the categorized env var listing.
- **`skill_manage` `list` action** — new read-only action that returns all existing skill names and descriptions, letting the LLM discover what exists before mutating. Requires no parameters.
- **`buildSkillManageToolDescription()`** — new dynamic description builder for the `skill_manage` tool that scans existing skills at tool-build time and injects their names + descriptions into the tool description (mirroring how the `skill` load tool surfaces available skills). The LLM no longer has to guess skill names when calling update/patch/delete.
- **`memory` `list` action** — new read-only action that returns current entries and char usage for a target, letting the LLM verify state before modifying. Bypasses permission checks (read-only).
- **`listEntries()` registry function** — new function returning formatted entries (numbered `[0]`, `[1]`, …) with char usage for a given memory target.
- **`formatEntriesForDisplay()` registry function** — new helper that formats raw `- bullet` entries as numbered `[index] text` strings for display in tool results and error messages.
- **`session_search` `list` action** — new read-only action that enumerates recent sessions in the workspace, returning each session's ID, title, message count, and last-updated timestamp. Lets the LLM discover what sessions exist before reading context from a specific one. Bypasses permission checks (metadata only, no message content exposed). Respects a `limit` parameter (default 10, max 20), excludes subagent child sessions, and marks the current session with `isCurrent: true`.
- **`session_search` read mode with optional `aroundMessageId`** — the `read` action (formerly read-around) now works with just a `sessionId`. When `aroundMessageId` is omitted, the tool automatically infers the latest message in the session as the anchor and returns the most recent context window. The result includes `anchorInferred: true` to signal that the anchor was auto-selected.
- **`SessionListEntry` type** — new exported interface for the `list` action results: `{ id, title, messageCount, updatedAt }`.

### Changed

- **Codex provider refactored** — no longer manages its own PKCE, localhost server, or token exchange. Uses the generalized `oauth-manager` for all OAuth operations. PKCE, state, and the localhost callback server are handled centrally.
- **`ConnectableProvider.connect()` return type** — now returns `ConnectResult` with `authorizationUrl`, `flowId`, `redirectStrategy`, and `redirectUri` instead of just `authorizationUrl`.
- **`ConnectOptions`** — renamed `serverBaseUrl` to `redirectStrategy` (type `OAuthRedirectStrategy`). The client now tells the server which redirect strategy to use instead of passing a base URL.
- **`ProviderStatusMessage` (WebSocket)** — now includes `flowId`, `redirectStrategy`, and `redirectUri` fields so the client can correlate pending flows.
- **Provider connect REST response** — `POST /api/providers/:id/connect` now returns `flowId`, `redirectStrategy`, and `redirectUri` alongside `authorizationUrl` and `status`.
- **`streamChat()` structured output** — when a model's capability is `prompt` mode and a `responseFormat` is requested, the schema is now appended to the system message and the result JSON is parsed from the streamed text via `extractJsonFromText()`. Native mode (the default) continues to use `Output.object()` with the AI SDK.
- **Model config schema** — `models.json` validator now accepts the optional `structuredOutput` capability object; all GLM and MiniMax models are preconfigured with `"mode": "prompt"`.
- **Categorized env var listing** — `listToolEnvVars()` now returns env vars classified by `source` (`'preset'`, `'tool'`, or `'custom'`). Preset vars carry their category, description, example, and link; tool-declared vars show which tools use them; custom vars are any remaining configured keys. `JEAN2_*` keys managed by the Credentials tab or server config are excluded from the custom bucket (except preset-prefixed keys like `JEAN2_GMAIL_*`).
- **`setToolEnvVar()` / `clearToolEnvVar()`** — now look up presets to populate `source`, `category`, `description`, `sensitive`, `example`, and `link` on the returned status, so preset variables keep their metadata across set/clear operations.
- **Workspace path resolution** — `expandPath()` in `routes/workspaces.ts` now resolves relative paths (e.g. `./project`, `../folder`) to absolute paths anchored at `process.cwd()`, and handles the bare `~` case. This fixes cross-platform correctness (macOS, Linux, multi-drive Windows) instead of storing relative paths verbatim.
- **`skill_manage` skill resolution** — update, patch, and delete now resolve skills case-insensitively against both the folder name and the frontmatter `name` field, instead of relying on an exact folder match. A skill created as `MyDebugFlow` (folder `mydebugflow`) can now be referenced as `MyDebugFlow`, `mydebugflow`, or by its frontmatter name regardless of folder name.
- **`skill_manage` error messages** — all "does not exist" errors now append the list of available skill names (e.g., `Available skills: shadcn, debug-flow`). The patch "oldString not found" error now hints to load the skill via the `skill` tool first to see the exact content, since exact string matching is otherwise a blind guess.
- **`skill_manage` update preserves skill name** — `update` now keeps the existing frontmatter `name` field instead of overwriting it with the sanitized slug, preventing unintended renames when the skill name differs from the folder name.
- **`memory` tool description** — now includes the character limits (user: 1500, workspace: 2500) so the LLM knows the budget upfront, and documents the `list` action for verifying current state before modifying.
- **`memory` error messages** — replace/remove "no match" errors now return entries as a numbered list (`[0] text`, `[1] text`, …) for easier targeting, and include a hint to use the `list` action. Memory-full errors now show char usage (`800/2500 chars`) and suggest merging related entries.
- **`MEMORY_GUIDANCE`** — system prompt guidance now includes the character limits and mentions the `list` action for verifying entries before modifying.
- **`session_search` tool description** — rewritten to describe three modes (list, search, read) and the typical workflow (list → read → search). Added `action` parameter (`list`, `search`, `read`) with smart defaults (search if query provided, read if sessionId provided).
- **`SESSION_SEARCH_GUIDANCE`** — updated to describe all three modes and the recommended workflow pattern.
- **`session_search` backward compatibility** — the tool still works without the `action` parameter: if `query` is present, it defaults to search mode; if `sessionId` is present, it defaults to read mode. Existing calls with just `query` or `sessionId + aroundMessageId` continue to work unchanged.

### Fixed

- **File browser path resolution in dev** — `GET /api/fs/browse` and `GET /api/fs/parent` now resolve relative paths against `homedir()` instead of implicitly using `process.cwd()`. In dev mode (`cd packages/server && bun --watch`), `process.cwd()` points to `packages/server`, causing relative paths sent by the folder picker to resolve to the wrong location. Uses `path.isAbsolute()` for cross-platform absolute path detection (Windows `C:\` and POSIX `/`).

- **Null-byte corruption in tool calls with GLM/Zhipu models** — the GLM API occasionally streams tool-call argument JSON with null-byte padding, causing file-editing tools (`edit`, `write-file`, `multiedit`) to receive truncated content with the remainder filled with null bytes. Since NUL is valid UTF-8, it passed through the entire decoding and parsing pipeline (`TextDecoderStream` → Zod `z.string()` → `JSON.parse`) untouched. `parseToolInput()` now recursively strips null bytes from all string values and keys in tool-call arguments, ensuring clean input reaches tool execution.

### Removed

- **Codex-specific localhost server** — the `Bun.serve()` on port 1455 that was hardcoded in `codex.ts` has been replaced by the generalized localhost server in `oauth-manager.ts`.
- **`handleCodexOAuthCallback` export** — replaced by the generalized `handleServerCallback` and `completeOAuthFlow` functions.
