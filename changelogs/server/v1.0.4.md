
### Fixed
- **Self-update now replaces the correct binary on Windows** — the updater was using a hardcoded path (`~/.jean2/bin/jean2.exe`) instead of the actual running binary location (`process.execPath`). If the binary was installed or running from a different location, the update would download successfully but replace the wrong file (or no file at all).
- **Self-update on Windows retries file replacement when the exe is locked** — the running `.exe` is locked by the OS and may not be released instantly after exit. The PowerShell replacer now retries `Move-Item` up to 10 times with 1-second delays instead of a single attempt after a 2-second sleep, giving Windows up to ~12 seconds total to release the file lock.
- **Post-update migration and daemon restart now run correctly on Windows** — the PowerShell command used `$LASTEXITCODE` to check if the binary replacement succeeded, but `$LASTEXITCODE` is only set by external executables, not PowerShell cmdlets like `Move-Item`. The check was always false, so migrate/restart steps never ran. Now uses a `$moved` flag set inside `try/catch` with `-ErrorAction Stop`.
