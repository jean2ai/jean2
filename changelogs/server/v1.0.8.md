### Added

- **Workspace additional paths** — workspaces now support `additionalPaths`: extra directories the agent has full access to without permission prompts. Paths are stored in a new `workspace_paths` DB table (`workspace_id`, `path`, `label`), threaded through the agent execution pipeline, and included in the system prompt so the LLM knows to use absolute paths for them.
- **`PATCH /api/workspaces/:id` extended** — now accepts `{ name?, additionalPaths? }`. Paths are validated to exist on disk before saving.
- **Path security updated** — `isPathWithinWorkspace`, `createPathHelpers`, and `getFilePreview` all check against the primary workspace path plus any additional paths.
- **Response format CRUD API** — new REST endpoints `GET/POST/PUT/DELETE /api/response-formats` with full create, read, update, delete support. Formats are stored in SQLite with JSON Schema definitions. Built-in formats no longer have delete protection — all formats are freely editable and deletable.
- **Structured output in messages** — when a response format is selected, the LLM's structured output is parsed from the stream result and attached to the final assistant message as `structuredOutput` (with `formatName`, `data`, `schema`). Stored in the `structured_output` column on the messages table.

### Fixed
- **Truncated tool results saved to malformed temp paths** — `truncateToolResult()` used `path.join(tmpdir(), 'jean2', '')` which produces a path without trailing separator, then string-concatenated the session ID directly, yielding paths like `/tmp/jean2abc123-...` instead of `/tmp/jean2/abc123-...`. The missing `/` meant `read-file` couldn't recognize the path as jean2-managed temp and unnecessarily prompted for permission. Now uses `path.join()` for both segments.

### Changed
- **Removed built-in format protection** — the DELETE endpoint no longer rejects `builtin-` prefixed IDs. All formats are treated equally.
