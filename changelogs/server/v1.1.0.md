### Fixed

- **Scheduler tool "create" action failing** — the `INSERT INTO scheduled_jobs` statement had 22 value placeholders for 21 columns (extra trailing `?`), causing a `SQLiteError: 22 values for 21 columns` that was silently swallowed, making the tool hang indefinitely. Removed the extra placeholder. Also wrapped the scheduler action dispatch in a try/catch so DB errors are returned to the agent as error results instead of hanging silently.

### Changed

- **Agents now use the live preconfig as single source of truth** — eliminated `agent.md` entirely. Previously, promoting a preconfig created a gray-matter `agent.md` snapshot containing a full copy of the preconfig (name, tools, model, systemPrompt, etc.) plus a `tagline`. This snapshot was used at runtime instead of the live preconfig, so edits to the original preconfig had no effect on agent behavior — a silent desync. Now the agent reads directly from the live preconfig via `getPreconfig()`:
  - Agent detection: checks for directory existence (`~/.jean2/agents/{id}/`) rather than `agent.md`
  - `getAgent()` reads the live preconfig and adds `hasHome`/`createdAt` from the directory
  - `getPreconfigOrAgent()` delegates directly to `getPreconfig()`
  - Removed `buildAgentMd`, `parseAgentMd`, `getAgentAsPreconfig`, `updateAgent`, and the `gray-matter` dependency from the agents module
- **Simplified promote** — `promotePreconfig()` no longer accepts tagline data. It just creates the directory structure (`skills/`, `home/`, `USER.md`, `MEMORY.md`) and the hidden home workspace.

### Added

- **Workspace preconfig settings validation** — the `PATCH /api/workspaces/:id` route now validates `settings.preconfigs.selectedIds` (array of strings or null) and `settings.preconfigs.defaultId` (string or null) to support workspace-specific preconfig selection.

### Removed

- **`PATCH /api/agents/:id` route** — the endpoint only updated tagline, which no longer exists.
- **`tagline`** — removed from the `Agent` type and all server-side agent handling.
