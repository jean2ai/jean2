### Added

- **Workflow tool** — new built-in tool that orchestrates parallel multi-agent work via a decompose → fan out → synthesize pattern. The agent calls `workflow({ prompt })` and the tool automatically decomposes the task into parallel subtasks (or accepts explicit subtasks), fans them out to leaf agents with a concurrency limit of 5, and synthesizes the results into one consolidated answer. Protects the main agent's context window — only the final synthesis is returned, not all intermediate results. Leaf agents are real visible sessions in the session tree. Supports optional `outputSchema` for structured final output and optional `leafPreconfigId` to force all leaf agents to a specific type. Gated behind a workspace-level setting (`workspace.settings.workflow.enabled`).
- **`core/workflow.ts`** — core workflow execution engine: `executeWorkflow()` with concurrency-limited async pool, `getWorkflowToolDefinition()` for AI SDK registration.
- **`core/workflow-decomposer.ts`** — Phase 1: isolated LLM call (via visible orchestrator session) that breaks a task into parallel subtasks. Filters to only allowed subagent types and sanitizes preconfigId assignments.
- **`core/workflow-synthesizer.ts`** — Phase 3: isolated LLM call (via visible orchestrator session) that combines leaf results. Supports optional structured output via schema-in-prompt (universal provider compatibility).
- **`core/workflow-orchestrator-session.ts`** — shared `runOrchestratorSession()` helper that wraps a single `generateText()` call in a visible child session with proper messages, parts, and broadcasts. Used by decomposer, synthesizer, and goal evaluator. Automatically attaches `structuredOutput` to the assistant message when the response contains valid JSON, so the client renders it as a structured card instead of raw text.
- **Goal mode** — new session-level mode that wraps the existing agent in a meta-loop. The user sets a completion condition, and the agent keeps working turn after turn until a separate evaluator confirms the condition is met (or a turn limit is reached). Inspired by Claude Code's `/goal` command. The agent is unaware it's in a goal loop — it receives continuation messages and keeps working. The evaluator decides when the goal is met by reading the conversation transcript (messages + tool outputs).
- **`core/goal-loop.ts`** — the outer loop: runs chat turn → evaluates → continues or stops. Detects interruption (user pressed Stop) and hard-stops the entire cycle. Goal state stored on `session.metadata.goal`.
- **`core/goal-evaluator.ts`** — isolated LLM call (via orchestrator session) that reads the transcript and judges completion. Returns structured `{ goalMet, reason, remainingWork }`. Builds continuation messages with evaluator feedback for the next turn.
- **Goal mode cancel support** — dedicated `AbortController` with 200ms polling detects session interrupts and hard-stops the goal loop. `ChatTurnResult.interrupted` flag propagated from `runSingleChatTurn` so the goal loop knows the turn was aborted and stops immediately — no evaluator runs, no next turn starts.
- **Preconfig slug generation** — `createPreconfig()` now generates a human-readable kebab-case slug from the preconfig name instead of a UUID. `"Code Reviewer"` → `code-reviewer`, with collision handling (`code-reviewer-2`, etc.). Prevents UUIDs from leaking into the LLM's tool definition enum, agent list, and subagent session titles.
- **Preconfig UUID migration** — `migrateUuidPreconfigs()` runs on server startup, scanning for UUID-named preconfig files and renaming them to slugs. Old UUIDs are recorded in `preconfig_aliases.json` so existing session references (`preconfig_id`, `agent_name`, `canSpawnSubagents` arrays) still resolve. Also normalizes `.json` preconfigs to `.md` format.
- **Preconfig alias resolution** — `getPreconfig(id)` now resolves through the alias map before looking up files. If an old session references a UUID that was migrated to a slug, the alias resolves it transparently.
- **`WorkspaceWorkflowSettings` type** — new workspace settings type (`{ enabled: boolean }`) gating the workflow tool. Follows the same pattern as memory/sessionSearch but without permission risk levels.
- **`WorkflowInput`, `WorkflowResult`, `WorkflowSubtask` SDK types** — shared types for the workflow tool input/output.
- **`GoalState`, `GoalStatus`, `GoalEvaluation` SDK types** — shared types for goal mode state and evaluation results.
- **`goalCondition` and `goalMaxTurns` on `ChatMessage`** — protocol fields for triggering goal mode from the client.

### Changed

- **Workflow tool is workspace-gated** — moved from the unconditional built-in tools list to a workspace-settings check in `buildAiSdkTools()`, matching the pattern used by memory and sessionSearch. The tool only appears when `workspace.settings.workflow.enabled` is true AND the preconfig allows spawning subagents. Default is off.
- **Goal mode branches in `handleChat()`** — `message-router.ts` detects `goalCondition` on incoming messages and routes to `runGoalLoop()` instead of `runSingleChatTurn()`. Goal state initialized on session metadata, turns run sequentially with evaluation between each, continuation messages injected when the goal is not yet met.
- **`runSingleChatTurn` detects interruption** — now checks if the last assistant message has `status: 'interrupted'` after the stream completes and returns `{ interrupted: true }` in the `ChatTurnResult`. The goal loop uses this to hard-stop.
- **`listMessagesWithParts` used by goal evaluator** — the evaluator builds a transcript summary from messages with parts (text + tool outputs) to judge goal completion. Tool outputs are truncated to 500 chars for context management.
- **Workspace settings validation** — `PATCH /api/workspaces/:id` now validates the `workflow` setting shape (`{ enabled: boolean }`) alongside existing memory/skills/sessionSearch validation.

### Fixed

- **Workflow decomposer suggested disallowed subagent types** — the decomposer called `listSubagentPreconfigs()` which returned ALL subagent preconfigs globally, but the parent session only allowed specific types. Every leaf agent was immediately rejected. Now passes `allowedSubagentIds` to the decomposer, filters the preconfig list, and post-sanitizes invalid `preconfigId` values to the first available allowed type.
- **Workflow all-failed case synthesized hallucinated output** — when all leaf agents returned errors, the workflow still ran synthesis which hallucinated an answer. Now bails early with a clear error listing all failure messages when `completedCount === 0`.
- **Workflow `completedCount` counted errors as successes** — the counter incremented before checking `result.error`, so failed agents were counted as completed. Now checks `result.error` before incrementing either counter.
- **Goal loop couldn't be cancelled** — `interruptManager.registerSession()` in the goal loop was overwritten by `streamChat()` on every turn, orphaning the goal loop's controller. Replaced with a dedicated `AbortController` + 200ms polling check for session interrupts.
