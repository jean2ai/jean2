### Changed

- **Reduced streaming persistence overhead** — assistant text and reasoning parts are no longer written to SQLite on every streamed delta. Streaming content is now kept live in memory and checkpointed to the database at a throttled interval, with a guaranteed final flush on completion, interruption, errors, and tool-call boundaries.
- **Deferred FTS work during active streams** — periodic streaming checkpoints now skip FTS synchronization. Search indexing still happens on final persisted state, avoiding repeated FTS rebuilds while the model is still generating.

### Fixed

- **Server responsiveness during fast streams** — mitigated event-loop stalls caused by per-token synchronous SQLite writes and repeated FTS updates in sessions where models stream quickly or produce many deltas.
- **Streaming durability at lifecycle boundaries** — pending text/reasoning content is flushed before stream completion, interruption handling, error handling, and tool execution transitions so persisted messages remain complete without requiring per-token writes.
- **Windows path handling in tool execution and file preview** — replaced `path.startsWith('/')` checks with `path.isAbsolute()` so Windows absolute paths (e.g., `C:\Users\...`) are correctly resolved instead of being incorrectly joined with the workspace path.
