### Added

- **Conflict-safe file editing service** (`services/fileMutations.ts`): `readEditableFile()` reads a UTF-8 file for editing with path traversal protection (lexical containment + realpath checks) and returns content with a SHA-256 revision hash. `saveFile()` implements optimistic concurrency control via SHA-256 revision comparison, throwing `ConflictError` (409) with the current content on mismatch. Writes are atomic (temp file + rename over real target), preserve existing file mode, and support a `force` flag.
- **File editing routes** (`routes/files.ts`): `GET /api/workspaces/:id/file` and `PUT /api/workspaces/:id/file` endpoints, with `saveFileSchema` Zod validation in `routes/schemas.ts`.
- **Workspace-scoped terminal reconnect validation** (`services/terminal/manager.ts`): `reconnectSession()` now accepts and validates `workspaceId`, returning a new `TerminalReconnectResult` type (`'connected' | 'not_found' | 'workspace_mismatch'`). Prevents a terminal session from one workspace being accessed via reconnect from another.
- **Terminal output replay on reconnect**: `TerminalManager.replaySession()` flushes the session's output buffer to the reconnecting WebSocket and sends an EXIT frame if the session has already exited, giving reconnecting clients the full terminal history.
- **OAuth `reauthRequired` state**: when a Gmail token refresh fails with `invalid_grant`, the server persists `reauthRequired = true` to the provider config, stops background refresh, and broadcasts a `provider.status` event with `connected: false, reauthRequired: true`. `getStatus()` returns this state, and startup skips background refresh for configs already marked. A successful refresh clears the flag.
- **`OAuthTokenRefreshError`** (`providers/oauth-manager.ts`): typed error class exposing `providerId`, HTTP `status`, OAuth `code`, and `description`. `refreshTokens()` now parses error response bodies as JSON to extract `error`/`error_description` fields instead of relying on raw body text.
- **OAuth completion broadcasts** (`providers/oauth-manager.ts`): after a successful token exchange, a `provider.connected` event is broadcast (with `connectedAt` and `accountId`). Error scenarios broadcast `provider.status` events, so clients receive real-time notifications without polling.

### Changed

- **WebSocket connections can participate in multiple active sessions**: a client connection now tracks a set of active session IDs instead of one active session. Creating, resuming, chatting in, or queueing a message for another session adds it to the connection without replacing participation in sessions that are already open. This allows multiple visible sessions to continue receiving queue, streaming, and control updates through the same client connection.
- **Session control cleanup covers every active session**: when a client disconnects, participant removal, controller grace periods, and pending takeover handling now run for every session associated with that connection. Secondary cleanup still removes stale participation records from any other session.
- **Codex provider uses Responses API** (`providers/codex.ts`): the ChatGPT OAuth provider now targets the OpenAI Responses API (`/v1/responses`) instead of Chat Completions, with `store: false` and `omitMaxOutputTokens: true` provider options.
- **Removed unmaintained providers**: deleted legacy dedicated provider modules. LLM providers (OpenAI, DeepSeek, OpenRouter, MiniMax, Zhipu) are now configured exclusively via the generic credential system rather than dedicated provider classes.
- **Client static serving** (`services/client-static-server.ts`): production-grade static file server with correct MIME type mappings (26 extensions), cache headers (`no-cache` for HTML/JS entry points, `immutable` for content-hashed assets), SPA navigation detection (falls back to `index.html` via `sec-fetch-mode` header), path traversal protection, HEAD support, and proper 405 handling for non-GET/HEAD methods.
- **OAuth callback page design**: success and error pages redesigned with a dark theme matching the jean2 UI, styled status icons, and HTML-entity escaping for error messages.

### Fixed

- **Codex context windows** (`config/models.json`): corrected context window values for Codex (ChatGPT) models to match actual API limits, preventing the server from advertising larger contexts than the provider supports.
- **OAuth callback pages rendering** (`providers/oauth-manager.ts`): all OAuth callback HTTP responses now include `Content-Type: text/html; charset=utf-8` and a `<meta charset="utf-8">` tag, fixing rendering issues with non-ASCII characters in error messages.
- **Client update reliability** (`services/client-launcher.ts`): client auto-updates are now atomic with staging/backup directories. The updated package is installed to a staging directory, validated for a usable CLI, then atomically swapped via rename. On failure, the previous version is restored from backup, preventing corrupted partial installs.
