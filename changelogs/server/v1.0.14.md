### Added

- **Gmail OAuth service provider** — new Gmail provider (`providers/gmail.ts`) that connects via Google OAuth 2.0 with PKCE. Registered as `kind: 'service'` (not `'llm'`), meaning it does not implement `createModel()`. Tools read the stored access token directly from `~/.jean2/providers/gmail.json`. Credentials configured via `JEAN2_GMAIL_CLIENT_ID` and `JEAN2_GMAIL_CLIENT_SECRET` env vars. Redirect URI is hardcoded to `http://localhost:1455/oauth/gmail/callback`.
- **Background token refresh for Gmail** — proactive timer that checks every 5 minutes and refreshes the Gmail access token before it expires (5-minute buffer). Starts automatically on server startup if Gmail was previously connected, and on successful OAuth completion. Eliminates the need for tools to handle token refresh logic or 401 retries. Uses an `isRefreshing` guard to prevent concurrent refresh races.
- **`GmailProviderConfig` SDK type** — extends `OAuthTokenConfig` with an optional `email` field (extracted from the Google `id_token` JWT during OAuth completion). Stored at `~/.jean2/providers/gmail.json`.
- **`kind` field on `ProviderDescriptor`** — new optional field (`'llm' | 'service'`) distinguishing LLM model providers from data/action service providers. Codex remains `'llm'` (implicit), Gmail is `'service'`.
- **`clientSecret` on `OAuthProviderConfig`** — optional field for OAuth providers that require a client secret in token exchange and refresh calls (Google, etc.). Included in both `exchangeCodeForTokens()` and `refreshTokens()` when present.
- **Multi-path localhost callback server** — the OAuth manager's localhost `Bun.serve()` listener now supports multiple callback paths on the same port. Gmail (`/oauth/gmail/callback`) and Codex (`/auth/callback`) coexist on port 1455 simultaneously. Uses a `Set<string>` of registered paths with reference-counted server lifecycle (`stopLocalServerForPath` replaces the old `stopLocalServers` that killed all servers at once).
- **`gmail-search` tool** — new external tool that searches and lists emails from the connected Gmail account. Supports Gmail search operators (`from:`, `subject:`, `is:unread`, `newer_than:`, `has:attachment`, `label:`, etc.). Returns message metadata (subject, from, to, date, snippet, labels, unread status) with optional full body via `includeBody`. Fetches detail for each message in parallel. Added to `repositoryv3.json` catalog.

### Changed

- **`createModel()` is now optional on `ConnectableProvider`** — service providers (Gmail) can omit it. `createModelForProvider()` throws a clear error (`"Provider does not support creating models (kind: 'service')"`) when called on a service provider, instead of crashing.

### Removed

- **`JEAN2_GMAIL_REDIRECT_URI` env preset** — removed from `env-presets.ts`. The redirect URI is hardcoded in `gmail.ts` and only needs to be registered in Google Cloud Console, not configured as an env var. Was showing an incorrect example value (`http://localhost:1455/auth/callback` — the Codex path) that would confuse users.
