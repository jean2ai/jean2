
### Changed
- **Refactored HTTP data fetching from manual `useEffect`/`useCallback` patterns to TanStack React Query** — added `@tanstack/react-query@5` with a centralized `QueryClient`, query key factory, and query/mutation hooks for all config panels (Tools, Models, Preconfigs, Prompts, Providers, MCP), file browsing (FileTree, FolderPickerDialog, file preview), session lists, and workspace CRUD. WebSocket event handlers now invalidate relevant query caches to keep them in sync with real-time state. Config mutations sync updated data back to the Zustand `serverDataStore` so components reading from either source stay consistent.

### Added
- **Inline error messages in the chat stream** — assistant messages that fail with an error (rate limit, server error, timeout, auth failure, context overflow, etc.) now render as styled inline error bubbles instead of empty messages. A new `ErrorMessageContent` component classifies errors by type and displays contextual icons, labels, and descriptions.

### Fixed
- **Error messages no longer appear as empty bubbles** — when an LLM request fails (rate limit, server error, timeout, etc.), the assistant message in the chat now shows the actual error information inline instead of rendering as an empty message.
- **Sound no longer plays on error messages** — the chat finish sound is now suppressed when the assistant message status is `error`, so users don't hear a false-positive "completion" chime on failures.
