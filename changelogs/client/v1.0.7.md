# v1.0.7

## Chat Transcript Scrolling

- **Fixed streaming-message jumps in free-look mode** — replaced the chat transcript's TanStack Virtual implementation with `@legendapp/list`, using `maintainVisibleContentPosition` so visible content stays anchored while a long assistant response continues streaming.
- **Improved streaming performance** — removed the manual virtualizer sizing/measurement pipeline, including row height tracking, ResizeObserver bookkeeping, and wrapper `translateY` layout. Streaming now feels smoother and avoids the previous scroll lag/forced-reflow behavior.
- **Preserved bottom-follow behavior** — the transcript still starts at the end, follows new output while auto-follow is enabled, and supports programmatic scroll-to-bottom from the follow control.
- **Improved free-look escape** — upward wheel/touch intent disables auto-follow immediately so users can reliably scroll away from the bottom even while messages are streaming quickly.
- **Removed unused dependency** — dropped `@tanstack/react-virtual` from the client package after migrating the transcript to `@legendapp/list`.

## Platform & VS Code Client

- **Added platform abstraction groundwork** — introduced shared platform hooks/adapters so the client can run consistently across browser, Electron, and VS Code-hosted environments.
- **Added capability-aware behavior** — platform capabilities now describe which host features are available, allowing the UI to adapt for VS Code-specific support without hard-coding environment checks throughout the app.
- **Prepared VS Code client variant** — added the platform/capabilities foundation needed for the VS Code version of the Jean2 client.

## Markdown Rendering

- **Kept markdown rendering simple during streaming** — reverted experimental streaming-specific markdown splitting/freezing. Markdown now renders predictably while the list handles visual scroll anchoring.
