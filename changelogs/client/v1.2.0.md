### Added

- **Toast notification system** — server errors, connection drops, and operation failures now surface as animated toast notifications (bottom-right) instead of being silently logged to `console.error`. Toasts respect dark/light mode and active color scheme (sunset, ocean, forest, etc.).
- **Loading states for all session operations** — fork, revert, edit, compact, delete, rename, and title regeneration now show visual feedback while in progress:
  - Fork and revert confirmation dialogs stay open with a spinner on the confirm button until the server responds
  - Compact button shows a spinner icon instead of the compact icon
  - Delete and regenerate title dropdown items show spinners and disable
  - Edit message shows an "Applying edit..." spinner after sending
- **Loading states for workspace operations** — create, delete, rename, paths, and settings all show feedback during HTTP requests. Delete confirmation dialog stays open with spinner. Error and success toasts on completion.
- **Loading states for minor operations** — pin/unpin message shows a spinner on the button, files panel refresh shows a spinning icon, bulk delete archived sessions dialog stays open with spinner.
- **Pending operations store** — Zustand store (`pendingOperationsStore`) tracks in-flight WebSocket operations. Operations are started on command send, cleared on success event or error, and cleaned up after a 60-second timeout with a "timed out" toast.

### Fixed

- **Tag input no longer loses focus while typing** — typing certain letters (r, a, c, d) in the tag input field inside the session dropdown menu would steal focus after each keypress, caused by Radix DropdownMenu's built-in typeahead matching menu item shortcuts. Keystrokes are now isolated from the menu's typeahead listener.
- **Operations no longer appear broken during server-side processing** — previously, fire-and-forget WebSocket commands (fork, revert, compact, etc.) showed no feedback between sending the command and receiving the response, making the app look frozen for slow operations like forking large conversations.
- **Connection drops clear pending operations** — when the WebSocket disconnects, all pending operations are immediately cleared with a "Connection lost" toast instead of hanging until the 60s timeout.
- **Themed toaster** — toast notifications now respect the app's dark/light mode and active color scheme. Previously the toaster rendered with hardcoded white backgrounds outside the theme provider.
- **Cmd+2 and Shift+Esc now work on the Changes tab** — the files panel keyboard shortcuts (Cmd+2 to focus, Shift+Esc to close) were broken when the Changes tab (grouped or flat mode) was active instead of the Project tab. `GitChangesView` now supports imperative focus, dispatching to the first file node or falling back to the container.
- **File preview scroll position no longer resets during session activity** — completed, failed, or interrupted file-mutating tool calls (edit, multiedit, write-file, apply-patch, shell) were broadly invalidating all file-related queries, causing the open file preview to refetch and remount its viewer (losing scroll position). File-list and Git-status queries are now invalidated separately from preview and Git-diff queries, so directory listings and status still update while the preview stays stable.

### Changed

- **`ConfirmationDialog` supports loading state** — new optional `loading` prop keeps the dialog open, disables all buttons, and shows a spinner on the confirm button. Used by fork, revert, delete, and bulk delete dialogs.
- **Theme-aware file icons** — file and folder icons in the file tree, changes view, autocomplete, path switcher, and folder picker dialogs now adapt to all 5 color schemes (neutral, ocean, forest, sunset, amethyst) and light/dark mode. Previously icons used hardcoded Tailwind colors that ignored the active theme. Color definitions are centralized in a shared `fileIcons.ts` utility with CSS custom properties registered per scheme/mode, replacing duplicated color maps across `FileTreeRow` and `GitChangesView`.
- **Grouped changes view renders full directory tree** — the Changes tab's grouped mode now represents every directory segment as an independently collapsible node instead of flattening all files under their top-level directory. For example, `src/components/ui/dialog.tsx` now expands through `src`, `components`, and `ui` as separate rows. Each directory starts collapsed, shows a recursive count of all changed files in its subtree, and can be expanded or collapsed independently. File rows display only the file name at the correct indentation depth while still selecting the full repository-relative path. Directories are sorted alphabetically before files at every level. Flat mode is unchanged.
