### Added

- **Goal mode UI** — new send-button mode toggle (🎯 Goal icon) in the chat input area. Clicking the icon opens a dropdown to switch between Chat and Goal modes. In Goal mode, the placeholder changes to "Type the completion condition..." and the send button turns amber. A max-turns stepper (− value +) replaces the browser number input. When a goal is active, the input is disabled and the placeholder shows "Goal active: {condition} (Turn X/Y)". The toggle is disabled (dimmed) while a goal is running to prevent starting a second one.
- **Unified Settings dialog** — merged the separate "Settings" and "Configuration" dialogs into a single "Settings" dialog with a grouped sidebar. **Preferences** section: Account, Appearance, Keybinds. **Server** section: Credentials, OAuth, Models, Prompts, Preconfigs, Formats, Environment. One gear icon replaces the two confusing buttons (Settings + Configuration). Eliminates the "which one do I click?" problem — everything is in one place.
- **Unified Workspace Settings dialog** — merged four separate workspace capability dialogs (Memory, Skills, Session Search, Workflow) into a single tabbed dialog with a sidebar. One button (Settings2 icon) replaces four toolbar buttons. Reduces workspace header clutter from 7 buttons to 4. Shared Save button persists all four capability settings at once.
- **Toolbar visual dividers** — added vertical dividers (`w-px h-5 bg-border`) in both the app header and workspace header expanded toolbars to visually separate navigation buttons from management buttons. In the app header: between Overview and Tools. In the workspace header: between Files and MCP Servers.

### Changed

- **AppHeader simplified** — replaced two buttons (Settings gear + Configuration sliders) with a single Settings button (gear icon). Tools now appears before Settings in both the expanded toolbar and dropdown menus.
- **WorkspaceHeader simplified** — replaced four capability buttons (Memory, Skills, Search, Workflow) with one Workspace Settings button (Settings2 icon). Reordered expanded toolbar: MCP, Permissions, divider, Additional Paths, Workspace Settings. Dropdown order: MCP, Permissions, divider, Additional Paths, Workspace Settings.
- **ConfigurationDialog renamed to "Settings"** — dialog title changed from "Configuration" to "Settings". Now accepts `apiToken`, `isConnected`, and `onLogout` props (previously on the deleted SettingsDialog). Sidebar shows "Preferences" and "Server" group labels.
- **`ConfigurationSection` type extended** — now includes `'account'`, `'appearance'`, `'keybinds'` alongside the existing server sections.
- **`setShowSettings` aliased to `setShowConfiguration`** — opening settings via the gear icon now opens the unified dialog.

### Removed

- **SettingsDialog.tsx** — deleted; preferences (Account, Appearance, Keybinds) extracted into panel components under `configuration/` and merged into ConfigurationDialog.
- **WorkspaceMemoryDialog.tsx** — deleted; extracted into `MemoryPanel.tsx`.
- **WorkspaceSkillsDialog.tsx** — deleted; extracted into `SkillsPanel.tsx`.
- **WorkspaceSessionSearchDialog.tsx** — deleted; extracted into `SessionSearchPanel.tsx`.
- **WorkspaceWorkflowDialog.tsx** — deleted; extracted into `WorkflowPanel.tsx`.
