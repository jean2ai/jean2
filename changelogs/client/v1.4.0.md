### Added

- **Web Push notifications**: installed web clients can enable system notifications for completed or failed assistant responses and permission requests from Appearance settings. Notification preferences and registration metadata persist across launches.
- **Notification deep links**: selecting a notification opens the matching server, workspace, and top-level session. App badges clear when Jean2 becomes visible or focused.
- **Scheduled job notification controls**: scheduled jobs can opt into completion and permission notifications per server, and enabled jobs show a notification indicator in the sidebar.
- **Overview workspace groups**: organize workspaces into named groups, assign or remove workspaces, select the active group per server, and reorder workspaces within a group. Groups persist in local storage and migrate existing quick connections on first use.
- **Assistant response forking**: assistant messages now expose a fork action that creates a new session from that response while leaving the original session unchanged.
- **Chat retry status**: active sessions show retry progress, countdowns, retry limits, and provider error details when a request is being retried.
- **Cache token usage display**: the token meter now reports cache-read, cache-write, and non-cached token counts alongside prompt, completion, total, and context-window usage.
- **Self-delegation setting**: preconfigurations can allow one immediate delegation to a fresh agent using the same preconfiguration.

### Changed

- **Custom PWA service worker**: the client now uses an inject-manifest service worker for precaching, push delivery, notification display, click routing, subscription recovery, and badge handling.
- **Cross-device notification suppression**: a visible, focused client displaying the completed session acknowledges the event so the server can avoid sending a redundant push to registered devices.
- **Notification registration recovery**: service-worker readiness now has a visible timeout, persisted registrations hydrate on startup, and foreground reconciliation restores server registration without duplicate immediate requests.
- **Terminal reconnect replay**: reconnecting terminals buffer replay output until the server confirms replay completion, then restore the terminal atomically without duplicating output or exposing a partially restored screen.
- **Scheduled sessions in Overview**: sessions created by scheduled jobs are hidden from Overview while remaining available through their normal workspace session lists.

### Fixed

- **Partial assistant responses**: provider failures during streaming no longer hide already-generated text, reasoning, and tool calls. The partial response remains visible with the error indicator.
- **User code-block formatting**: pasted user messages preserve line breaks inside fenced code blocks instead of doubling them during markdown rendering.
- **Queued message presentation**: queued user messages no longer receive the inverted-message formatting intended for completed user messages.
- **Markdown table visualization**: markdown tables now retain horizontal scrolling and consistent borders.
- **Compaction status timeout**: the client acknowledges server-side compaction as soon as it starts, preventing the pending operation from timing out while compaction continues.
- **Preconfig duplication refresh**: duplicated preconfigurations now refresh the local query and store state after creation.
