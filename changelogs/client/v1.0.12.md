### Added

- **Manual paste OAuth fallback** — the OAuth connection panel now shows a paste input field after initiating a connection. Users can copy the redirect URL from the browser (which fails for remote servers) and paste it back to complete the OAuth flow. Supports full URLs, query strings, or bare authorization codes.
- **`useCompleteOAuth` mutation hook** — new TanStack Query mutation that posts `{ flowId, code, state, redirectUri }` to `POST /api/oauth/callback` and invalidates provider queries on success.
- **`completeOAuth()` SDK REST method** — new method on `ProvidersRestNamespace` for submitting OAuth authorization codes to the server.
- **Environment panel in Configuration** — new "Environment" section in the Configuration dialog listing all env vars by source (preset, tool, custom) with category, description, example, and documentation links. Preset variables (e.g. Gmail OAuth) show rich metadata from the server's `ENV_PRESETS`.
- **Structured output mode editor** — the Models panel now exposes a dropdown to set a model's `structuredOutput.mode` (`default/native`, `native (json_schema)`, or `prompt-based`), with a badge showing the active mode on each model card.
- **`ConfigurationSection` UI store state** — new Zustand slice (`configurationSection`, `setConfigurationSection`) that drives the Configuration dialog's active section, enabling deep-linking/opening to a specific tab.
- **`.dialog-scrollbar` CSS** — new thin, theme-aware scrollbar style for dialog content areas (Tools, Settings, Configuration), matching the chat transcript aesthetic.

### Added

- **Nested field support in visual format builder** — the Response Formats visual editor now supports recursive nesting. Object fields expand to show their nested properties, and List (array) fields expand with an item-type selector that can be set to Object to define per-element properties. This enables building complex schemas like "list of objects with name and value" entirely in the visual builder, without needing raw JSON. Nesting works to arbitrary depth (arrays of objects of arrays, etc.).
- **Schema round-trip for nested structures** — editing an existing format with nested schemas no longer silently drops nested `properties` or `items`. `schemaToFields`/`fieldsToSchema` now recursively parse and serialize the full nested tree (objects with `children`, arrays with `items`), preserving round-trip fidelity.

### Changed

- **OAuth connection flow** — the connect flow now stores pending auth state (`flowId`, `redirectUri`, `authorizationUrl`) and shows a richer UI with the auth URL, copy button, and manual paste input, instead of just displaying the URL as a link.
- **`useConnectProvider` mutation** — now accepts `{ providerId, redirectStrategy }` instead of `{ providerId, serverBaseUrl }`. No longer passes `getStoredServerUrl()` — the server determines the redirect URI from the registered OAuth config.
- **Provider list query invalidation** — `useCompleteOAuth` invalidates `queryKeys.config.providers.all` on success so the UI immediately reflects the connected state.
- **Configuration dialog redesign** — replaced the 6-column horizontal tab bar with a vertical sidebar (desktop) / Select dropdown (mobile). Switched from `ScrollArea` to a flex-grown `overflow-y-auto` content area with `min-h-0` for reliable scrolling. Dialog now uses a fixed `85dvh` height and mobile-safe centering. All panel padding tightened to `p-3 sm:p-4`.
- **Settings dialog redesign** — same vertical sidebar + Select dropdown pattern applied. Converted to controlled section state via `useState`. Keybinds list now wraps on narrow screens.
- **Tools dialog simplified** — removed the single-tab wrapper and `ScrollArea`; the panel now scrolls directly inside the dialog's `dialog-scrollbar` container.
- **`DialogContent` mobile centering** — dialog now centers within the safe area (below notch/status bar) using `env(safe-area-inset-top)` so the close button stays tappable on mobile PWA. Degrades to plain centering on desktop.
- **Tools env filter** — the Tools panel now only shows env vars with `source === 'tool'`; preset and custom vars are managed in the Configuration → Environment panel.
- **Tool card responsive badges** — tool list badges collapse text labels under `sm:` so they fit on mobile widths.
- **Mobile-safe viewport units** — Settings/Configuration dialogs use `dvh` units (`h-[85dvh]`) for correct sizing on mobile browsers.

### Fixed

- **Question tool selection bug** — clicking any option in a `multi_select` question inside a form could select ALL options when options shared duplicate `value` fields. Selection state is now tracked by option index instead of `option.value`, so duplicate values no longer cause cross-selection. React keys also use the index to prevent DOM reuse issues.
- **Question tool single/multi visual distinction** — `single_select` and `multi_select` questions inside forms now look visually different: single-select shows radio circles, multi-select shows checkboxes with a "Select all that apply" / "N selected" hint and max-limit disabling.
- **Folder picker path desync** — `FolderPickerDialog` now syncs `currentPath` from the server's resolved path (`directoryData.currentPath`). Previously the initial empty path was resolved to `homedir()` server-side, but the client state stayed `''`, causing `join('', folderName)` to produce relative paths. This broke "Add existing folder" in dev mode (paths resolved relative to the server's CWD instead of the home directory shown in the UI).
- **Format builder duplicate field names on save** — adding multiple nested properties (e.g. inside a List → Object) without renaming each one first caused all fields to collide on the default name `field_1` and silently overwrite each other during save, so only the last property persisted. The "Add Property" buttons now generate sequential unique default names (`field_1`, `field_2`, …) based on the current child count, and `fieldsToSchemaFragment` additionally de-duplicates any sibling name collisions with an auto-suffix regardless of what the user types.

### Removed

- **`getStoredServerUrl()` usage in OAuth panel** — the client no longer sends the server URL during provider connect. The redirect URI is determined server-side from the registered `OAuthProviderConfig.redirectUri`.
- **`ScrollArea` in dialogs** — Tools, Settings, and Configuration dialogs no longer use `ScrollArea`; replaced by native `overflow-y-auto` with the `dialog-scrollbar` class for reliable flex-column scrolling.
