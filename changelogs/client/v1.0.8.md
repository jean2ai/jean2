# v1.0.8

## Structured Response Formats

- **Added response format configuration** — new "Formats" tab in the Configuration dialog with full CRUD for response formats. Create, edit, and delete formats from a visual schema builder.
- **Added visual schema builder** — field-by-field editor with name, type (Text, Number, Yes/No, List, Object), description, and required toggle. No need to write JSON Schema by hand. Raw JSON mode still available via toggle for power users.
- **Added structured response visualization** — when a response format is active, the parsed output renders as a human-readable view instead of raw JSON: booleans show as ✓ Yes / ✗ No, primitive arrays as pill badges, objects as labeled sections, nested data with visual indentation. Raw output is accessible via a collapsible toggle.
- **Added response format selector in chat input** — pick a format from a dropdown before sending a message. Selected format is highlighted with the format name. Wider dropdown with descriptions shown below format names.
- **Fixed format selector stale data** — selector now shares the TanStack Query cache with the config panel, so newly created/edited formats appear immediately without reloading.

## Workspace Additional Paths

- **Added additional paths dialog** — new "Additional Paths" option in the workspace header 3-dot menu and workspace selector 3-dot menu. Opens a dialog to add/remove directories the agent can access alongside the primary workspace.
- **Added folder picker integration** — "Add Path" button opens the existing folder picker dialog to browse and select directories.
- **Fixed dialog not showing existing paths** — the additional paths dialog now correctly syncs its state when opened, instead of relying on `useState` initialization that only runs once.

## Configuration

- **All formats can be deleted** — removed built-in format protection. Every format (including defaults) can be edited or deleted for full control.
