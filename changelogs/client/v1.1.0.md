### Added

- **Agent home auto-redirect** — navigating to the Agents section (`/agent/`) immediately drops into the first agent's workspace instead of showing a separate list page. If no agents exist, shows a minimal empty state with a "Promote Preconfig" button. Gets users working in their agent workspace faster — no intermediate management screen.
- **Inline agent management in sidebar** — the AgentSwitcher popover (the dropdown in the sidebar header) now handles all agent management inline, matching the WorkspaceSwitcher UX pattern:
  - **Switch agents** — click any agent in the searchable dropdown list
  - **Promote** — "Promote Preconfig" action at the bottom opens the promote dialog
  - **Demote** — per-agent dropdown menu (⋯) with "Demote Agent" + confirmation dialog
  - Active agent shows a checkmark, just like workspaces
- **Workspace-specific preconfig selection** — new "Preconfigs" tab in Workspace Settings (under General). Select a subset of preconfigs per workspace and choose the default for new chats. The New Chat button, keyboard shortcut (Ctrl/Cmd+K), and `createSessionInWorkspace` all respect the workspace default. The chat header preconfig dropdown is filtered to the workspace-selected set. When none are selected (or an empty selection), all primary preconfigs remain visible — backward compatible with existing workspaces.
- **Searchable tool selector in preconfig editor** — the tool enable/disable UI in the preconfig editor (Settings > Preconfigs) is now a searchable command palette with sorted tools, inline descriptions, and removable chips for selected tools. Scales to 25+ tools without a massive unsorted badge wall.

### Changed

- **Agents now use the live preconfig as single source of truth** — previously, promoting a preconfig to an agent created a frozen `agent.md` snapshot that diverged from the original preconfig. Editing the preconfig had no effect on its agent instance. Now the agent reads directly from the live preconfig at runtime, so edits propagate immediately. No more desync.
- **Consistent preconfig icons** — promoted agents show the Bot icon (primary color) while normal preconfigs show the Cog icon, matching the chat header dropdown across all UI surfaces (workspace preconfig panel, agent switcher, preconfig editor).
- **Agent promote no longer requires a tagline** — the promote dialog no longer asks for a tagline. Just pick a preconfig and click Promote.

### Removed

- **AgentList page** — the full-page card grid with "Open Home" buttons is gone, replaced by the inline AgentSwitcher experience that drops you straight into a workspace.
- **Tagline** — the `tagline` field has been removed from agents across the SDK, server, and client UI. It added complexity without meaningful value.
- **EditTaglineDialog** — deleted along with tagline.
- **useUpdateAgent hook** — deleted (it only updated tagline).
- **PromoteOptions/UpdateAgentOptions types** — deleted from the SDK.
