## v1.0.10

### Fixed
- **Windows PATH/executable resolution**: Removed the PowerShell wrapper that was causing `ENOENT` errors when spawning `powershell.exe`. The shell tool now invokes `cmd.exe` directly, relying on the server's environment (which now includes `PATHEXT`, `SYSTEMROOT`, `COMSPEC`, and other Windows-critical variables) for native executable resolution.
- **cmd.exe path resolution**: `cmd.exe` is now resolved to an absolute path via `ComSpec` → `SystemRoot\System32` → `C:\Windows\System32` fallback chain before spawning. This prevents `ENOENT: uv_spawn cmd.exe` when the server holds a stale or stripped PATH snapshot (e.g. daemon-launched with a filtered environment before the env whitelist fix is deployed).

### Changed
- Simplified Windows shell invocation from a multi-step PowerShell `Start-Process` wrapper (~40 lines) to a direct `cmd.exe /d /s /c <command>` call. This eliminates a fragile intermediary and the `ENOENT: uv_spawn powershell.exe` failure mode.
